# L2 Reference 6×6 FAIL 원인 분석 및 해결방안

**작성일:** 2026-03-18  
**문제:** ΔP_sim = 0.774 Pa vs ΔP_theory = 3.819 Pa (오차 80%)

---

## 1. 근본 원인: LBM 비압축성 한계 위반

솔버 버그가 아닙니다. **문제는 물리적 ΔP가 LBM이 표현할 수 있는 범위를 초과한 것입니다.**

### 1.1 LBM 압력-밀도 관계

LBM에서 압력은 밀도로 표현됩니다:

```
P = ρ × c_s² (격자 단위, c_s² = 1/3)
ΔP_phys = Δρ_lbm × p_scale
p_scale = ρ_phys × (dx/dt)² / 3
```

현재 설정의 p_scale:

```
dx = 2.0e-4 m, dt = 3.6e-5 s
dx/dt = 5.556 m/s
p_scale = 0.746 × 5.556² / 3 = 7.675 Pa (격자 Δρ 1.0당)
```

### 1.2 요구 밀도 변화량 비교

| 검증 | ΔP_theory | 요구 Δρ_lbm | Δρ/ρ | LBM 허용? |
|------|-----------|-------------|------|-----------|
| L1 빈 덕트 | 0.076 Pa | 0.010 | **1.0%** | ✅ 문제없음 |
| L2 6×6 | 3.819 Pa | 0.498 | **49.8%** | ❌ 완전 위반 |

**6×6 채널의 ΔP를 정확히 표현하려면 밀도가 입구 대비 50% 차이나야 합니다.** LBM의 비압축성 근사는 Δρ/ρ < ~1%일 때 유효하고, 10%만 넘어도 오차가 급격히 커집니다. 50%는 LBM이 물리적으로 표현할 수 없는 영역입니다.

이것이 시뮬 ΔP가 이론의 1/5밖에 안 나오는 이유입니다. 솔버가 "밀도를 더 올릴 수 없어서" 압력차가 포화됩니다.

### 1.3 L1은 왜 통과했나

L1(빈 덕트)의 ΔP = 0.076 Pa → Δρ = 1.0%로, 비압축성 범위 안이기 때문입니다. 솔버 자체는 정상 작동합니다.

### 1.4 Gyroid에서도 같은 문제 예상

```
Gyroid a=5mm, t=0.3 (대표 케이스):
  ε ≈ 0.5, u_channel ≈ 0.56 m/s, Dh ≈ 2mm
  ΔP_theory ≈ 11 Pa → Δρ = 1.43 → 143%
  → L2보다 더 심각
```

**이 문제를 해결하지 않으면 BO 전체가 의미 없습니다.**

## 2. 해결방안 비교

| 방안 | 원리 | 장점 | 단점 |

## 3. 권장: 방안 A + B 병행

### 3.1 방안 A — 즉시 실행 (L2 재검증)

u_in을 **1/5로 축소**하여 L2 검증을 수행합니다.

```
u_in_low = 0.2778 / 5 = 0.0556 m/s  (GHSV 2,000 h⁻¹ 상당)
u_channel_low = 0.449 / 5 = 0.0898 m/s
Re_low = 39.1 / 5 = 7.82
ΔP_theory_low = 3.819 × (1/5)² = 0.153 Pa
Δρ_lbm = 0.153 / 7.675 = 0.020 → 2.0% (LBM 허용 범위)
```

**이 조건에서 ΔP 오차 < 5%가 나오면, 솔버가 올바르다는 것이 확인됩니다.** 6×6 형상 + bounce-back + 다채널 유동이 정확히 해석되는지를 검증하는 것이 L2의 목적이므로, 유속을 낮춰도 검증 의미는 동일합니다.

검증 후 논문에는 "LBM 비압축성 한계를 고려하여 저유속 조건에서 HP 해석해와 비교 검증하였다"로 기술하면 됩니다. 이건 LBM 논문에서 일반적인 접근입니다.

### 3.2 방안 B — BO 파이프라인용 (솔버 수정 필요)

BO에서 Gyroid 케이스를 GHSV 10,000으로 돌리려면, **주기경계 + 체적력 구동** 방식이 필요합니다. 이 방식에서는:

```
Z방향 주기 BC: F[x,y,0] ↔ F[x,y,NZ-1]
체적력 g: 각 스텝마다 f_eq에 g 항 추가
ΔP = ρ_lbm × g × NZ × (dx²/dt²) × ρ_phys  (격자 변환)
```

밀도 구배 없이 유동을 구동하므로 Δρ ≈ 0이 되어 비압축성 위반이 원천적으로 사라집니다.

**이건 솔버 수정이 필요하므로 L2 검증 후 별도 단계로 진행합니다.**

## 4. CV 40% 문제

6×6 마스크에서 CV가 40%인 것도 확인이 필요합니다.

```
내부 공간: 117 voxels
내부 격벽: 5개 × 5 voxels = 25 voxels
채널용: 117 - 25 = 92 voxels
92 / 6 = 15.33 → 정수로 나누어지지 않음!
```

**일부 채널은 15 voxels(3.0mm), 일부는 16 voxels(3.2mm)가 됩니다.** 이게 CV 40%의 원인입니다. HP에서 ΔP ∝ 1/Dh⁴이므로, 1 voxel 차이(6.7%)가 ΔP에서는 (1.067)⁴ - 1 ≈ 30% 차이를 만듭니다.

**해결:** 격자 수를 조정하여 균등 분할이 되도록 합니다.

```
방법 1: NX=NY를 조정
  외벽 5 + 채널×6 + 격벽×5 + 외벽 5
  = 10 + 6a + 25  (a = 채널 voxel 수)
  a=15이면 NX = 10+90+25 = 125
  a=16이면 NX = 10+96+25 = 131

방법 2: 격벽 두께를 4 voxel로 조정
  내부 격벽: 5 × 4 = 20
  채널용: 117 - 20 = 97 / 6 → 역시 안 나눠짐

방법 3: NX=NY=131, dx=0.2mm → 26.2mm 덕트
  내부: 131-10 = 121
  격벽: 25
  채널: 96/6 = 16 (정수!) ✅
```

**NX=NY=131로 변경하면 채널이 16 voxels씩 균등 분할됩니다.**
