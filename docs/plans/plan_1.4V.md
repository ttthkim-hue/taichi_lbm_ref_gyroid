## 1. 빈 덕트 L1 본검증 실행 (크리티컬 패스)

**현재:** 스크립트 준비 완료, 미실행 상태. 이것이 전체 프로젝트의 병목.

**실행:** `scripts/run_l1_empty_duct_plan13v.py` → GPU, 127×127×550, 수렴까지 (~70분)

**이론값 재검산 (실행 전 확인):**

| 항목 | 값 |
|------|-----|
| 내부 유로 | 25.4mm 전체에서 벽 5voxel(1mm) × 양쪽 → 내부 23.4mm? 혹은 25.4mm 그대로? |
| Dh | **벽 처리에 따라 달라짐 — 코드의 빈 덕트 voxel 마스크를 확인하여 실제 내부 유로 폭 산출** |
| f·Re | 56.91 (정사각 단면) |
| ΔP_theory | Dh 확정 후 직접 재계산 |

> ⚠️ 계획서의 ΔP ≈ 0.032 Pa는 Dh=25.4mm 기준. 벽 5voxel이 내부로 잡히면 Dh가 줄고 ΔP가 커짐. **voxel 마스크의 유체 영역 실제 치수를 먼저 확인할 것.**

**PASS 기준:** 시뮬 ΔP vs 이론 ΔP 오차 < 5%

**FAIL 시 점검 순서:**
1. ΔP 측정면(Z_in, Z_out) 위치가 발달 구간 밖인지
2. 스텝 수 부족 (수렴 로그에서 ΔP가 아직 변하고 있는지)
3. Q_in vs Q_out 차이 > 1%이면 BC 문제
4. 총 질량 드리프트 > 0.1%이면 bounce-back 또는 BC 누수

## 2. run_with_logging에 early stopping 구현 확인

**문제:** 수렴 판정 로직은 있으나, 실제로 루프를 break하는지 보고서에 미명시.

**확인:** ΔP 변화율 3회 연속 < 0.1% 시 **루프 즉시 종료** 되는지 코드 확인. 안 되어 있으면 추가.

**이유:** BO에서 케이스마다 5,000~80,000스텝 편차 발생. early stopping 없으면 모든 케이스가 max_steps(100,000)까지 돌아 시간 낭비.

## 3. Gyroid 커널 t 범위 검증

**문제:** 현재 구현 `solid = 1 if |φ| < t` (sheet 타입)에서 t ≤ 0이면 solid가 생성 안 됨. 계획서의 t ∈ [−0.5, 0.5]와 불일치 가능.

**할 일:** 아래 (a, t) 조합으로 커널 실행 후 공극률(ε) 테이블 생성

| a (mm) | t 값 | 확인 항목 |
|--------|------|-----------|
| 5.0 | 0.0 | ε ≈ 1.0 (벽 없음) 이면 sheet 타입 정상 |
| 5.0 | 0.1 | ε 계산, 벽 두께 ≥ 3 voxels 여부 |
| 5.0 | 0.3 | ε ∈ [0.35, 0.65] 범위 내인지 |
| 5.0 | 0.5 | ε < 0.35 제약 위반 여부 |
| 3.0 | 0.3 | 소형 셀에서 벽 연결성 유지되는지 |
| 8.0 | 0.1 | 대형 셀에서 벽이 너무 얇지 않은지 |

**산출물:** `(a, t) → (ε, min_wall_voxels)` 매핑 테이블. BO 탐색 범위 확정에 사용.

**BO 설계 변수 범위 재조정:** 테이블 결과에 따라 t의 실효 범위를 [t_min, t_max]로 축소. t ≤ 0 영역이 무의미하면 t ∈ [0.05, 0.5] 등으로 변경.

## 5. ΔP 측정면 — BO 단계 대비 메모

현재 측정면(buf+5 / NZ-1-buf-5)은 빈 덕트에서는 문제없음. 그러나 Gyroid BO 시에는 **촉매층 시작/끝 위치 기준**으로 재정의해야 함.

도메인 구조가 "버퍼 + 촉매층 + 버퍼"라면:
- Z_in = 촉매층 시작 + 5 cells
- Z_out = 촉매층 끝 − 5 cells

이건 BO 파이프라인 구축 시 반영. 지금은 메모만.
