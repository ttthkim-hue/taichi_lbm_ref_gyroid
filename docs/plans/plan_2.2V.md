# L2 FAIL 근본 원인 분석 및 해결 지시서 — plan_2.2V

**작성일:** 2026-03-18  
**문제:** L2-A 오차 79%, L2-B 오차 83%, K(A) vs K(B) 30배 차이

---

## 1. 핵심 발견: 원인 2개

### 원인 1: Guo forcing의 (1-s/2) 팩터 오류 [L2-B, 확실도 95%]

L2-B에서 u_sim/u_theory = 0.0761/0.449 = **0.169**

MRT에서 Guo forcing의 올바른 구현은 moment별로 다른 s값을 사용해야 합니다:

```
보존 moment (ρ, jx, jy, jz): s = 0   → 팩터 = 1 - 0/2 = 1.0
응력 moment:                  s = 1/τ → 팩터 = 1 - 1/(2τ) = 0.16
```

만약 코드가 **모든 moment에 단일 τ를 적용**하면:

```
(1 - 1/(2×0.595)) = 1 - 0.840 = 0.160
```

이론 유속 × 0.160 = 0.449 × 0.160 = **0.0718** ≈ 시뮬 0.0761 ✓

**운동량에 적용되는 체적력이 1/6로 줄어든 것.** 이 숫자 일치는 우연이 아닙니다.

### 원인 2: u_in이 채널 유속인지 표면 유속인지 혼동 [L2-A, 확실도 80%]

velocity inlet BC에서 u_in은 **Z=0의 유체 셀에만** 적용됩니다. 6×6의 경우 Z=0의 유체 셀이 곧 채널 입구이므로:

```
시뮬 실제: u_channel = u_in = 0.01234 m/s
이론 가정: u_channel = u_in / ε = 0.01234 / 0.6295 = 0.0196 m/s
```

이론이 u_in을 superficial velocity로 가정했지만, 시뮬에서는 pore velocity로 적용된 것입니다. 이 차이만으로 오차의 약 37%를 설명합니다.

나머지 오차(추가 ~3배)의 가능 원인:

```
(a) ΔP 측정에서 고체 셀 포함 여부 → Δρ가 ε배 희석
(b) 이론 Dh vs 실효 Dh (bounce-back 벽면 위치)
(c) (a)+(b) 복합
```

## 2. 수치 검증

### 2.1 이론값 재검산 (독립 계산)

6×6 채널, Dh=3.2mm, HP Fanning:

```
ΔP = 2 × f_F·Re × μ × u_ch × L / Dh²
   = 2 × 14.227 × 2.626e-5 × u_ch × 0.107 / (0.0032)²
   = 7.474e-4 × u_ch × 0.107 / 1.024e-5
   = 7.81 × u_ch
```

| u_ch 가정 | ΔP_theory | vs ΔP_sim(0.032) |
|-----------|-----------|------------------|
| 0.0196 (u_in/ε) | 0.153 Pa | 오차 79% |
| 0.01234 (u_in 직접) | 0.096 Pa | 오차 67% |
| 0.01234 + ε 희석 | 0.061 Pa | 오차 48% |

이론값 교정만으로는 0.032에 도달하지 않으므로, **ΔP 측정 로직 자체에도 문제가 있을 가능성이 높습니다.**

### 2.2 K 이론값 독립 산출

```
Darcy: K = ε × Dh² / (2 × f_F·Re)
     = 0.6295 × (0.0032)² / (2 × 14.227)
     = 0.6295 × 1.024e-5 / 28.454
     = 2.27e-7 m²
```

| K | 값 | vs K_theory | 비고 |
|---|-----|-------------|------|
| K_theory | 2.27e-7 | — | 해석적 |
| K_sim(A) | 1.08e-6 | 4.8배 큼 | u_in을 superficial로 쓰면 ε 보정 필요 |
| K_sim(A)×ε | 6.80e-7 | 3.0배 큼 | ε 보정해도 여전히 큼 |
| K_sim(B) | 3.62e-8 | 0.16배 | (1-1/2τ) 팩터와 일치 |
| K_sim(B)/0.16 | 2.26e-7 | **1.00배** ✅ | Guo 팩터 보정 시 이론과 **정확히** 일치 |

**K_sim(B)를 Guo 팩터(0.16)로 나누면 이론값과 정확히 일치합니다.** 이것은 원인 1이 맞다는 결정적 증거입니다.

## 3. 진단 지시 (코드 확인, 시뮬 불필요)

### 진단 A: Guo forcing 구현 확인 [최우선]

`solver/taichi_lbm_core.py`에서 Guo forcing 커널을 찾아서 아래 확인:

```python
# 올바른 MRT Guo forcing:
# moment space에서 (I - S/2) 적용 시
# s[0] = 0 (밀도) → factor = 1.0
# s[3] = 0 (jx)   → factor = 1.0  
# s[5] = 0 (jy)   → factor = 1.0
# s[7] = 0 (jz)   → factor = 1.0  ← 체적력이 여기 들어감
# s[9~13] = 1/τ   → factor = 0.16

# 잘못된 구현 (BGK 방식):
# F_i = (1 - 1/(2*tau)) × w_i × [...]  ← 모든 방향에 동일 τ 사용
```

**확인할 것:**
1. forcing 항에서 `(1 - 0.5/tau)` 같은 단일 τ 사용 여부
2. MRT의 S 대각에서 보존량(s₀,s₃,s₅,s₇=0)이 forcing에도 반영되는지
3. 운동량 moment에 대한 forcing factor가 1.0인지

### 진단 B: ΔP 측정 커널 확인

```python

rho_sum = 0.0
count = 0
for i, j in range(NX, NY):
    if solid[i, j, z_plane] == 0:  # 유체만
        rho_sum += rho[i, j, z_plane]
```

**확인할 것:**

## 4. 수정 지시

### 수정 1: Guo forcing — MRT 호환 [최우선]

```python
# MRT에서 Guo forcing의 올바른 구현:
# 1. 물리 공간에서 source term S_i 계산
# 2. moment 공간으로 변환: S_m = M × S
# 3. (I - S_diag/2) 곱하기 — S_diag는 19개 moment별 relaxation rate
# 4. 다시 물리 공간으로: F = M⁻¹ × (I - S_diag/2) × S_m
# 5. f_i += F_i

# 핵심: step 3에서 보존 moment(인덱스 0,3,5,7)의 s=0이므로 factor=1.0
# 참고: Guo et al., PRE 65, 046308 (2002)
#        Lallemand & Luo, PRE 61, 6546 (2000)
```

> ⚠️ 코드 예시는 의도 전달용. 기존 M, inv_M, S_dig 구조에 맞춰 구현할 것.
