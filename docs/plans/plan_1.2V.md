## 1. 물리 파라미터 교차검증

### 1.1 격자-물리 변환 일관성 검증

계획서에 명시된 파라미터들의 내부 정합성을 수식으로 검증한다.

**주어진 값:**
- dx = 0.2 mm = 2.0 × 10⁻⁴ m
- τ = 0.595
- ν_phys = 3.52 × 10⁻⁵ m²/s
- u_in_phys = 0.2778 m/s
- Ma = 0.087 (계획서 기재)

**LBM 동점성계수:**

```
ν_lbm = (τ − 0.5) / 3 = (0.595 − 0.5) / 3 = 0.03167
```

**시간 스텝 (dt) 역산:**

```
ν_phys = ν_lbm × dx² / dt
→ dt = ν_lbm × dx² / ν_phys
     = 0.03167 × (2.0×10⁻⁴)² / (3.52×10⁻⁵)
     = 0.03167 × 4.0×10⁻⁸ / 3.52×10⁻⁵
     = 3.60 × 10⁻⁵ s
```

**격자 유속 (u_lbm) 역산:**

```
u_lbm = u_in_phys × dt / dx
      = 0.2778 × 3.60×10⁻⁵ / 2.0×10⁻⁴
      = 0.0500
```

**Mach Number 검증:**

```
Ma = u_lbm / c_s = u_lbm × √3 = 0.0500 × 1.732 = 0.0866 ≈ 0.087 ✅
```

> **결론: 계획서의 τ, Ma, dx 값은 상호 정합적이며 오류 없음.**

### 1.2 물성값 검증 (200°C 건조공기)

| 파라미터 | 계획서 값 | NIST 참조값 (473K, 1atm) | 판정 |
|----------|-----------|--------------------------|------|
| ν | 3.52 × 10⁻⁵ m²/s | ~3.5 × 10⁻⁵ m²/s | ✅ 정합 |
| ρ | 0.746 kg/m³ | ~0.746 kg/m³ (이상기체) | ✅ 정합 |
| μ = ν × ρ | 2.626 × 10⁻⁵ Pa·s | 2.57~2.63 × 10⁻⁵ Pa·s | ✅ 허용 범위 |

> **물성값 정확함. μ = ν × ρ 계산도 올바름.**

### 1.3 GHSV → 유속 변환 검증

```
GHSV = 10,000 h⁻¹
촉매층 길이 L_cat = 100 mm = 0.1 m
u_in = GHSV × L_cat / 3600 = 10000 × 0.1 / 3600 = 0.2778 m/s ✅
```

> **GHSV 변환 정확.**

## 2. 수치 안정성 및 건전성 분석

### 2.1 τ = 0.595의 안정성

τ가 0.5에 가까울수록 수치 불안정이 증가한다.

| 항목 | 평가 |
|------|------|
| BGK에서의 안정성 | ⚠️ τ = 0.595는 BGK에서 불안정 가능성 있음 |
| MRT에서의 안정성 | ✅ MRT는 relaxation rate를 개별 제어하므로 τ ≈ 0.6에서도 안정적 |
| 권장 조치 | MRT relaxation matrix (S_i) 설정값을 명시하고, ghost moment 의 relaxation rate를 1.0 이상으로 설정할 것 |

**핵심 주의사항:** MRT-LBM을 사용한다고 명시되어 있으므로 τ = 0.595는 허용 범위이나, **코드 내 MRT S-matrix가 정확히 구현되어 있는지 반드시 확인해야 함.** 만약 실제 코드가 BGK(SRT)로 구동되고 있다면 τ = 0.595에서 Gyroid 벽면 근처에서 음의 분포함수(negative distribution function)가 발생하여 시뮬레이션이 발산할 수 있음.

### 2.2 질량 보존 (Mass Conservation)

LBM에서 질량 보존 위반이 발생할 수 있는 주요 지점:

| 위험 요소 | 심각도 | 설명 및 대책 |
|-----------|--------|-------------|
| **Velocity Inlet BC** | 🔴 높음 | Zou-He 또는 Regularized BC 사용 시, 밀도가 자유변수로 남음. 정상상태에서 입구/출구 질량유량이 일치하는지 매 N스텝마다 검증 필요 |
| **Pressure (Open) Outlet BC** | 🟡 중간 | Anti-bounce-back 또는 Convective outlet 사용 시 질량이 점진적으로 누적될 수 있음. ρ_outlet 모니터링 권장 |
| **Bounce-back on Voxel Surface** | 🟡 중간 | Standard halfway bounce-back은 질량 보존적이나, 계단형(staircase) 근사에 의한 유효 벽면 위치 오차 존재 |
| **복셀화 누수 (Leak)** | 🔴 높음 | STL이 watertight하지 않거나, voxelization 해상도가 얇은 벽을 놓치면 고체 내부로 유체가 침투 |

**필수 검증 항목:**
1. 매 1000 스텝마다 `mass_total = Σ ρ(x)` 모니터링 → 드리프트 < 0.1% 확인
2. 정상상태에서 `Q_in ≈ Q_out` (유량 밸런스) 확인
3. Gyroid 벽 두께가 voxel 기준 최소 3셀 이상인지 확인 (dx=0.2mm, 벽 두께 ≥ 0.6mm)

### 2.3 Voxelization 정밀도

```
벽 두께 1.0 mm / dx 0.2 mm = 5 voxels (벽 두께 방향)
```

5 voxels는 벽면 해석에 최소한으로 충분하나, 다음을 주의:

| 설계 변수 조합 | 위험 |
|----------------|------|
| t → +0.5 (두꺼운 벽) | 공극률 ε < 0.35 제약 위반 가능 |
| t → −0.5 (얇은 벽) | 실제 벽 두께가 1mm 미만이 되어 voxel이 끊어질 수 있음 (연결성 상실) |
| a → 3mm (작은 셀) | 단위 셀당 15 voxels → 곡면 해상도 부족, 계단 효과 심화 |

**권장:** BO 실행 전 (a, t) → 벽 두께 매핑 함수를 사전에 만들어, 벽 두께 < 1.0mm인 조합은 BO 탐색 공간에서 제외할 것 (infeasible constraint).

### 2.4 비압축성 근사 유효성

Ma = 0.087 < 0.1 이므로 비압축성 근사 유효 조건을 충족한다. 그러나 Gyroid 내부 좁은 채널에서 국소 유속이 평균의 3~5배까지 증가할 수 있으므로:

```
국소 Ma_max ≈ 0.087 × 5 = 0.435 → ⚠️ 비압축성 한계 초과 가능
```

**대책:** 시뮬레이션 후처리 시 `max(|u_lbm|)` 값을 확인하여 국소 Ma < 0.3을 검증할 것. 초과 시 격자를 세분화하거나 τ를 조정해야 함.

### 3.2 연산 시간 추정

**RTX 5060 Ti 예상 성능:**

Blackwell 아키텍처 기반으로, Taichi D3Q19 MRT-LBM에서 약 **100~200 MLUPS** (Million Lattice Updates Per Second)를 예상한다. 보수적으로 **120 MLUPS**를 기준으로 산정.

**정상상태 수렴까지 필요한 스텝 수:**

```
유동 통과 시간 (flow-through time):
  t_ft = L / u_in = 0.11 / 0.2778 = 0.396 s

수렴 기준: ~5 flow-through times (Gyroid 복잡 형상 고려)
  t_conv = 5 × 0.396 = 1.98 s

필요 타임스텝:
  N_steps = t_conv / dt = 1.98 / 3.60×10⁻⁵ = 55,000 steps
```

**단일 케이스 소요시간:**

```
시간/스텝 = 8.87M / 120M = 0.0739 s/step
총 시간 = 55,000 × 0.0739 = 4,065 s ≈ 68 min/case
```

**STL 생성 + Voxelization 시간:** ~2~5 min/case (CPU)

| 항목 | 소요시간 (추정) |
|------|----------------|
| 단일 LBM 케이스 | **~70 min** |
| STL 생성 + Voxelization | ~3 min |
| 후처리 (ΔP, S_v, CV 산출) | ~2 min |
| **단일 BO iteration** | **~75 min** |

## 4. 코드 파이프라인 잠재적 문제점

### 4.1 STL → Voxel 파이프라인

| 문제 | 심각도 | 설명 |
|------|--------|------|
| STL watertightness | 🔴 높음 | OpenSCAD로 생성한 Gyroid STL이 non-manifold edge를 포함할 경우 voxelization이 내부/외부를 잘못 판별. Trimesh의 `is_watertight` 검사 필수 |
| Voxel 경계 정합 | 🟡 중간 | STL 좌표계와 LBM 도메인 좌표계가 정확히 일치하는지 확인 필요. 오프셋 1~2 voxel 오차가 ΔP에 큰 영향 |
| 덕트 벽 + Gyroid 중첩 | 🟡 중간 | 25.4mm 덕트 내벽(1mm)과 Gyroid 구조가 올바르게 Boolean 결합되는지 확인 |

### 4.2 Taichi LBM 솔버

| 문제 | 심각도 | 설명 |
|------|--------|------|
| MRT 구현 검증 | 🔴 높음 | D3Q19 MRT의 transformation matrix M과 relaxation matrix S가 정확히 구현되었는지 확인. 문헌(Lallemand & Luo, 2000)의 행렬과 대조 필수 |
| Bounce-back 구현 | 🟡 중간 | Halfway bounce-back이 solid→fluid 방향에서만 적용되는지 확인. 양방향 적용 시 질량 보존 위반 |
| 경계조건 호환성 | 🟡 중간 | Zou-He velocity inlet + pressure outlet 조합에서 모서리/꼭짓점 노드 처리가 누락되기 쉬움 |
| 수렴 판정 기준 | 🟡 중간 | 현재 계획서에 수렴 판정 기준 (예: `‖u^(n+1) − u^(n)‖ / ‖u^(n)‖ < 10⁻⁶`) 이 명시되지 않음 |

### 4.3 후처리

| 문제 | 심각도 | 설명 |
|------|--------|------|
| ΔP 측정 위치 | 🟡 중간 | 촉매층 입구/출구에서 측정할 때, 입구 발달 구간(약 5~10mm)을 제외하고 측정해야 정확 |
| S_v 산출 방법 | 🟡 중간 | Voxel 기반 표면적 계산은 Marching Cubes 대비 과대 추정(최대 ~30%). STL 표면적을 직접 사용하는 것이 더 정확 |
| CV 측정 단면 | 🟡 중간 | CV를 어느 단면에서 측정하는지 명확히 정의 필요 (출구 단면? 촉매층 중앙?) |

## 5. Bayesian Optimization 설정 검토

### 5.1 설계 공간

| 항목 | 평가 |
|------|------|
| a ∈ [3, 8] mm | ✅ 적절. 다만 a=3mm에서 단위 셀당 voxel 수가 15개로 형상 해상도 부족 우려 |
| t ∈ [−0.5, 0.5] | ⚠️ 범위가 넓어 비현실적 형상 생성 가능. 공극률/벽 두께 제약으로 일부 방어되지만, BO가 infeasible region을 탐색하며 시간 낭비 가능 |

**권장:** BO 전에 (a, t) → (ε, wall_thickness) 매핑 테이블을 만들어, 제약 조건 위반 영역을 사전에 마스킹할 것.

### 5.2 목적함수 3개 → 실용성

3-목적함수 Pareto 최적화는 BoTorch의 qNEHVI로 가능하나:

| 우려 | 설명 |
|------|------|
| 표본 효율 | 3-objective BO는 2-objective 대비 2~3배 많은 표본이 필요. 40~50 iterations 이상 권장 |
| Pareto 시각화 | 3D Pareto front는 논문에서 직관적 표현이 어려움 |
| CV의 독립성 | Gyroid의 대칭적 특성상 CV가 (a, t)에 크게 민감하지 않을 수 있음 → 사실상 2-objective로 축소될 가능성 |

**권장:**
- 1차로 f₁(S_v) vs f₂(ΔP)의 2-objective BO를 수행하여 논문 핵심 결과 확보
- CV는 Pareto 최적해에 대한 사후 분석으로 보고
- 이렇게 하면 BO iterations을 30회 내외로 줄일 수 있어 시간 절약

## 7. 논문 투고 관련 권장사항

### 7.1 검증 체계 (Verification & Validation)

SCI 논문에서 요구하는 최소 검증 수준:

| Level | 내용 | 현재 상태 |
|-------|------|-----------|
| L1 | 빈 덕트 ΔP vs Hagen-Poiseuille (정사각 단면 보정 포함) | ⚠️ 미확정 |
| L2 | Reference 6×6 ΔP vs 이론/해석 | ❌ 미완 |
| L3 | 격자 독립성 (GCI) | ❌ 미계획 |
| L4 | 질량 보존 검증 (시간 추이) | ❌ 미계획 |

**L1~L4 모두 완료해야 리뷰어 대응 가능.**

### 7.2 PiAM 저널 적합성

Progress in Additive Manufacturing (PiAM)에 투고 시, 리뷰어가 중시할 포인트:

1. **3D 프린팅 공정과의 연결성** — BJT 해상도 제약이 설계 변수 범위에 실제로 반영되어 있음을 강조
2. **실험 검증** — 시뮬레이션만으로는 impact가 제한적. 최소 1~2개 Pareto 최적해에 대한 실험 ΔP 측정 데이터가 있으면 논문 수용률 대폭 향상
3. **기존 문헌 대비 novelty** — TPMS+LBM 조합 자체는 이미 다수 발표됨. "STL→Voxel 자동 파이프라인 + Multi-objective BO"가 차별화 포인트

| 2 | Voxelization 누수 (얇은 벽 관통) | 질량 보존 위반, ΔP 비정상 | 모든 (a,t) 조합에서 벽 두께 ≥ 3 voxels 사전 검증 |
| 3 | 빈 덕트 L1 검증 미완 | 논문 리젝 사유 | 즉시 Hagen-Poiseuille 비교 실행 및 문서화 |
| 4 | 격자 독립성 미계획 | 논문 리젝 사유 | 3-level GCI 스터디 추가 |
| 5 | 국소 Ma 초과 (좁은 채널) | 비압축성 가정 위반 | 후처리 시 max(Ma) 모니터링 |
| 6 | 3-objective BO의 표본 비효율 | 시뮬 시간 초과 | 2-objective (S_v, ΔP)로 축소 권장 |

## 9. 수정된 타임라인 제안

GPU: RTX 5060 Ti 16GB 기준, 24시간 연속 가동 가정.

| 주차 | 작업 | 산출물 |
|------|------|--------|
| 1주 | 코드 통합, MRT 구현 검증, L1 빈 덕트 검증 완료 | 검증 PASS 문서 |

## 10. 결론

계획서 v1.1의 물리 파라미터(τ, Ma, dx, 물성값)는 내부 정합성을 갖추고 있으며 수치적으로 건전하다. RTX 5060 Ti 16GB로 현재 격자(8.87M nodes)를 충분히 처리할 수 있고, Fine 격자(20.9M nodes)까지 수용 가능하다.

**즉시 해결해야 할 핵심 사항 3가지:**

1. **MRT 구현 실제 확인** — 코드가 진짜 MRT인지, BGK(SRT)인지 확인
2. **L1 빈 덕트 검증 완료 및 문서화** — 이것 없이 다음 단계 진행은 위험
3. **격자 독립성 스터디 추가** — 논문 필수 요소

이 세 가지가 확보되면, 나머지 파이프라인은 계획서대로 진행 가능하다.
