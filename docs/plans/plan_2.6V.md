# 작업지시서 — plan_2.6V (주기BC 단축 도메인 + 검증 + BO 경로)

**작성일:** 2026-03-19  
**핵심:** NZ=550 → NZ=50(단위셀 2개)로 축소. 검증·BO 모두 단축 도메인 사용.

---

## 0. 왜 줄여도 되는가

주기BC는 "무한히 긴 관의 일부분"을 시뮬하는 것이다. Z 양 끝이 연결되므로, **동일한 구조가 무한 반복**된다고 가정한다. 따라서:

- **K(투과율):** 단위 체적당 성질. 1셀이든 22셀이든 동일한 K가 나와야 함.
- **S_v(비표면적):** 단위 체적당 표면적. 도메인 길이에 무관.
- **ΔP:** K로부터 역산. ΔP = u·μ·L / K. L은 임의로 설정 가능.

**LBM 다공성 매질 논문에서 주기BC + 1~2 단위셀은 표준 방법이다.**

| 도메인 | NZ | 노드 수 | 수렴 스텝 | 케이스 소요 | BO 50회 |
|--------|-----|---------|-----------|------------|---------|
| 현재 (22셀) | 550 | 9.4M | 50,000+ | ~140분 | ~117시간 |
| **2셀 (권장)** | **50** | **0.86M** | **~3,000** | **~2분** | **~100분** |
| 1셀 | 25 | 0.43M | ~2,000 | ~1분 | ~50분 |

**검증과 BO 모두 NZ=50(2 단위셀)을 사용한다.** 1셀은 경계 효과가 미미하게 남을 수 있으므로, 안전하게 2셀을 기본으로 한다.

## 1. 도메인 설정

### 1.1 NZ 산출 규칙

```
NZ = round(2 × a / dx)   (단위셀 2개)

예시:
  a = 5mm, dx = 0.2mm → NZ = 2 × 5 / 0.2 = 50
  a = 3mm, dx = 0.2mm → NZ = 2 × 3 / 0.2 = 30
  a = 8mm, dx = 0.2mm → NZ = 2 × 8 / 0.2 = 80
```

### 1.2 NX, NY는 기존 유지

```
NX = NY = 131  (균등 분할, 외벽 5 voxel 포함)
```

### 1.3 Gyroid 커널 수정

주기BC에서는 **버퍼 구간(빈 유로)이 불필요**하다. 전체 Z에 Gyroid를 배치한다.

```python
# 기존 (velocity inlet용):
# Z ∈ [0, 5mm]: 버퍼 (빈 유로)
# Z ∈ [5, 105mm]: Gyroid
# Z ∈ [105, 110mm]: 버퍼

# 수정 (주기BC용):
# Z ∈ [0, NZ×dx]: 전체 Gyroid (버퍼 없음)
# 외벽: XY 경계 5 voxel만
```

## 2. 즉시 실행: 도메인 길이 무관성 검증

**목적:** NZ=50과 NZ=550에서 K가 동일한지 확인. 이것이 통과하면 짧은 도메인의 정당성이 증명됨.

### 2.1 설정

Gyroid a=5, t=0.3, Network, 주기BC + 체적력 g=5e-6

| 케이스 | NX×NY | NZ | 노드 수 | 비고 |
|--------|-------|-----|---------|------|
| Short | 131×131 | 50 | 0.86M | 2 단위셀 |
| Long | 131×131 | 550 | 9.4M | 22 단위셀 (현재) |

### 2.2 스크립트

`scripts/verify_nz_independence.py` 신규 작성:

```python
"""
NZ 무관성 검증: NZ=50 vs NZ=550에서 K 비교.
동일 g에서 K 차이 < 5%이면 PASS.
"""
# 두 NZ에 대해 순차 실행
# 각각:
#   1. 솔버 생성 (주기BC, g=5e-6)
#   2. Gyroid 커널 (a=5, t=0.3, 버퍼 없이 전체 Z에 배치)
#   3. 수렴까지 실행 (max 10,000 스텝, Q 변화율 < 0.1%)
#   4. K 산출
# 비교: |K_short - K_long| / K_long < 5%
```

**예상 시간:** Short ~5분 + Long ~30분 = ~35분

### 2.3 PASS 기준

```
|K_short - K_long| / K_long × 100 < 5%  → 단축 도메인 PASS
```

## 6. BO 파이프라인도 단축 도메인

### 6.1 BO 도메인 설정

```
NX = NY = 131 (고정)
NZ = round(2 × a / dx)  (a에 따라 가변)
mode = periodic_body_force
g_lbm = 고정값 (예: 5e-6)
max_steps = 5,000 (수렴 시 early stop)
```

### 6.2 BO 목적함수

| 목적 | 산출 방법 | 도메인 의존? |
|------|-----------|-------------|
| f₁: S_v 최대화 | voxel 표면적 / 체적 | ❌ 무관 |
| f₂: K⁻¹ 최소화 | Darcy: K = u·μ·L / ΔP | ❌ 무관 (K는 내재 성질) |

**두 목적함수 모두 도메인 길이에 무관하므로, 단축 도메인으로 BO 전체를 수행할 수 있다.**

### 6.3 BO 예상 시간

```
케이스당: ~2분 (NZ=50 기준)
초기 탐색 15회: ~30분
반복 최적화 35회: ~70분
총 BO 시뮬: ~100분 (~1.5시간)

전체: 검증 1시간 + BO 1.5시간 = 약 2.5시간
```

### 6.4 BO 결과 → 물리 ΔP 환산

BO에서 K를 구한 뒤, 실제 운전 조건의 ΔP는 Darcy로 환산:

```
ΔP = u_in × μ × L_catalyst / K

예: K=1e-7 m², u_in=0.2778 m/s (GHSV 10k), μ=2.626e-5, L=0.1m
  → ΔP = 0.2778 × 2.626e-5 × 0.1 / 1e-7 = 7.3 Pa
```

논문에서는 이 환산값으로 ΔP를 보고한다.
