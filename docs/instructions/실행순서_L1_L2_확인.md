# 실행 순서 및 L_measure·L2 확인 사항

**반영 지시:** Shah & London 수식 확인, L_measure 로그, L2 u_in/ L_measure, 실행순서.

---

## 1. L_measure (L1)

- **의미:** Shah & London 수식에 넣는 길이는 **측정면 간 거리**이어야 함 (덕트 전체 110 mm 아님).
- **계산:** `z_in = buf+5 = 7`, `z_out = nz-1-buf-5 = 542` → `L_measure = (542 - 7) × 0.0002 = 0.107 m = 107 mm`.
- **로그:** `run_l1_empty_duct_plan13v.py` 실행 시 상단에 다음 한 줄이 출력됨:
  ```
  [L_measure 검증] z_in = 7, z_out = 542  →  L_measure = (542 - 7) × 0.2 mm = 107.000 mm (덕트 전체 110 mm 중 측정 구간)
  ```

---

## 2. L2 확인 사항

### 2.1 Inlet BC

- **시뮬레이션 inlet BC:** `u_in = 0.2778 m/s` (그대로 사용). `TaichiLBMWrapper(..., u_in_phys=u_in)`.
- **u_channel = 0.449 m/s:** 이론 ΔP·Re_ch 계산용만 사용. 시뮬에서는 채널 유속 가속이 자동으로 일어남.
- 스크립트에 `시뮬 inlet BC: u_in = 0.2778 m/s` 로그 추가됨.

### 2.2 HP ΔP 이론 길이

- **L_measure 사용:** L1과 동일하게 `L_measure = (z_out - z_in) × dx` (측정면 간 거리).
- 덕트 전체 110 mm가 아님. 6×6는 Le=6 mm라 발달유동 보정은 없지만, **비교 기준 길이는 L1과 통일**.
- 스크립트에 `L_measure = ... mm (HP 이론 길이)` 로그 추가됨.

## 3. 실행 순서

1. **L1:** `python scripts/run_l1_empty_duct_plan13v.py`  
   - 약 70분. 수렴 후 [L1 검증 — 발달유동 상관식] 블록에서 PASS/FAIL 확인.
2. **L1이 FAIL이면 L2 실행하지 않음.** 원인 점검 후 L1 재실행.
3. **L1 PASS 후:** `python scripts/run_l2_ref6x6.py`  
   - 약 70분. [L2 검증 — Reference 6×6 채널] 블록에서 PASS/FAIL 확인.
4. **L1·L2 모두 PASS** → plan_1.4V §3 (Gyroid t 범위 검증)으로 진행.

## 4. 로그 파일

- L1 실행 시: `run_l1_empty_duct_plan13v.py` 표준출력이 `run_l1_log.txt`에 tee로 저장됨 (실행 디렉터리 기준).
- L1 완료 후 해당 파일 끝부분에서 `[L1 검증 — 발달유동 상관식]` 블록 및 판정 확인.
