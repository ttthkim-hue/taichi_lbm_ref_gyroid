## 3. 물리적 분석

### 3.1 압력강하 (ΔP)

- **시뮬 ΔP**: 5.983 Pa.
- **정상 유동 가정 Poiseuille (f·Re=56.9)**  
  - Re = U·D_h/ν.  
  - U ~ 5.6 m/s, D_h = 0.0252 m, ν = 1.616e-5 → Re ≈ 8,700.  
  - f = 56.9/Re, ΔP_th = f·(L/D_h)·(1/2)·ρ·U² → **약 2~2.5 Pa** 수준.
- **해석**: 시뮬 ΔP(5.98 Pa)가 이론 추정보다 큼.  
  - 원인 후보: (1) inlet/outlet BC로 인한 압력 구배·가속, (2) ρ_in > ρ_out 에 따른 비정상 압력 분포, (3) 목표 u_phys(0.28 m/s)와 실제 유동(Uin/Uout ~5.6 m/s) 간 단위/스케일 불일치 가능성.

### 3.2 질량 보존 (Flux 균형)

- **Inlet 질량 유량** (비례): ρ_in × Uin ≈ 0.682 × 5.62 ≈ **3.83**  
- **Outlet 질량 유량** (비례): ρ_out × Uout ≈ 0.227 × 5.70 ≈ **1.29**  
- **불균형**: inlet > outlet → **도메인 내 밀도/질량 누적** 경향.
- **판정**: **질량보존 미충족**. Outlet BC 또는 압력(밀도) 경계 처리 재검토 필요.

### 3.3 속도 크기 (목표 대비)

- 목표: u_phys = **0.2778 m/s**.  
- 로그 상 Uin/Uout ≈ **5.6 m/s** → 목표의 약 20배.
- 가능한 원인: (1) **물리 단위 변환** (lattice→phys) 계수 차이, (2) **압력 구배**에 의한 가속, (3) **RAMP/BC**로 인한 유입량 과대.  
- **정밀 검증**을 위해서는 동일 조건(동일 ν, ρ, L, D_h, u_in)에서 이론/OpenLB와 단위를 맞춘 비교가 필요.

### 3.4 수치 안정성

- **τ = 0.55**, **Umax_lb = 0.15** clamp 적용 → KBC 안정 구간 유지.
- **res** ~ 2.8e-3로 발산 없이 완료.
- **결론**: 60,000 스텝 구간에서는 **수치적으로 안정**하게 종료됨.

## 4. OpenLB·이론과의 비교

### 4.1 OpenLB 빈 덕트 결과 (참고)

| 출처 | ΔP [Pa] | K [m²] | u_avg [m/s] | converged |
|------|---------|--------|-------------|-----------|
| result_duct_v4 | 0 | 0 | 0 | false (발산) |
| result_duct_forum | 0.0359 | 1.17e-4 | 1.46 | false |
| result_caseB | 0.0359 | 2.11e-5 | - | true |

- OpenLB **verify_duct_v4** (126×126×550, dx=0.2 mm, u=0.2778)는 **발산**으로 기록됨 (result_duct_v4.json).
- Forum/case 계열은 형상·물성·경계가 상이하여 **직접 수치 비교는 어렵다**.

### 4.2 Taichi vs OpenLB (동일 형상 가정 시)

- **형상**: 동일 `verify_duct_nowall` (126×126×550) 사용.
- **차이점**  
  - **유체**: Taichi = 공기(ν=1.616e-5), OpenLB = 케이스별 상이(예: ν=3.52e-5, ρ=0.746).  
  - **경계조건**: Taichi = 속도 inlet + Neumann outlet, OpenLB = InterpolatedPressure + pressure ramp 등.  
- **결과**: OpenLB 쪽은 동일 격자에서 발산한 반면, Taichi는 **60k 스텝 정상 종료 + ΔP·U 기록**까지 완료.  
- **직접 ΔP/K 비교**: 물성·BC가 다르므로 **동일 조건으로 재설정 후 비교**하는 것이 타당함.

### 4.3 이론값 (정사각 덕트, 동일 L·D_h)

- L = 0.11 m, D_h = 0.0252 m, f·Re = 56.9.  
- **u = 0.2778 m/s**, ν = 1.616e-5, ρ = 1.2 → Re ≈ 433, f = 56.9/433,  
  **ΔP_th ≈ 0.21 Pa** (공기 기준).  
- **u = 5.6 m/s** 가정 시: Re ≈ 8,700, **ΔP_th ≈ 2~2.5 Pa**.  
- 시뮬 5.98 Pa은 위 두 경우보다 크므로, **BC/단위/밀도장** 재점검 필요.

## 5. 결론 및 권장 사항

### 5.1 실행 결론

| 항목 | 판정 | 비고 |
|------|------|------|
| **완료 여부** | ✅ 정상 종료 | 60,000 스텝 완료, 로그·CSV 생성 |
| **수치 안정성** | ✅ 안정 | res ~ 2.8e-3, τ·Umax_lb 제한 유지 |
| **질량 보존** | ❌ 미충족 | inlet flux > outlet flux, ρ_in > ρ_out 지속 |
| **OpenLB와 직접 비교** | ⚠️ 제한적 | 물성·BC 상이, OpenLB 동일 형상은 발산 |
| **이론 ΔP와의 일치** | ❌ 불일치 | 시뮬 5.98 Pa vs 이론 0.2~2.5 Pa 수준 |

### 5.2 권장 사항

1. **경계조건·단위 정합**  
   - OpenLB와 동일 물성(ν, ρ) 및 동일 u_in, L, D_h를 사용하는 설정 파일을 두고, Taichi에서 **같은 물리 조건**으로 재실행.  
   - Inlet/outlet에서 **질량 유량 균형**이 나오도록 BC(압력/밀도 처리) 조정.

2. **질량보존 지표 정량화**  
   - 매 스텝 또는 구간별 **전역 ρ 합**, **inlet/outlet flux**를 계산해 `mass_drift`, `flux_imbalance`를 로그에 기록.  
   - 보고서/검증 기준에 “flux 불균형 < x%” 조건 추가.

3. **후처리 및 비교 자동화**  
   - `postprocess.py`로 **u_avg, K_m2, ΔP_Pa, CV** 계산 후 `result_taichi_empty_duct.json` 저장.  
   - `compare_taichi_openlb.py`로 OpenLB result JSON과 **동일 키** 기준 비교 (가능 시 동일 물성 케이스만).

4. **필드 덤프**  
   - 수렴 구간(예: step 60,000)에서 **u_z**(및 필요 시 u_x, u_y), inlet/outlet **압력(또는 ρ)** 을 npy/json으로 덤프해 프로파일·이론(Poiseuille)과 비교.

5. **문서화**  
   - 위 항목 반영 후 `docs_Taichi_LBM_ref_gyroid_report.md`에 **빈 덕트 절**을 채우고, “OpenLB 설정 그대로 재검증” 목표 충족 여부를 **조건 정리 + 판정(충족/미충족)** 으로 명시.
