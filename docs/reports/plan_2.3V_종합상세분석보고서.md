## 2. Phase 1: 코드 리뷰 결과

### 2.1 진단 A — p_scale 산출 경로

| 확인 항목 | 결과 |
|-----------|------|
| p_scale 계산식 | `cs2_phys = (1/3) * (dx/dt)²`, `p_scale = rho_phys * cs2_phys` → **/3 1회만** |
| ΔP 산출 | `get_delta_p_pascal = get_delta_p_lattice(z_in, z_out) * p_scale` → **추가 /3 없음** |
| run_with_logging 내 ΔP | `dP = self.core.get_delta_p_pascal(...)` 그대로 사용 → **추가 변환 없음** |

**결론:** p_scale에 c_s² 이중 적용(후보 1)은 **코드상 없음**.

### 2.2 진단 B — _slice_rho_mean

| 확인 항목 | 결과 |
|-----------|------|
| 유체만 평균 | `solid[i,j,z_in]==0`, `solid[i,j,z_out]==0` 일 때만 ρ 합산·카운트 |
| count | 유체 셀 수 (c0, c1). 전체 NX×NY 아님. |
| 외벽 | z_in = buf+5, z_out = nz-1-buf-5 → 도메인 내부 |

**결론:** ρ 평균은 **유체 전용**. 고체 포함(후보 3) 희석은 구현상 없음.

### 2.3 진단 C — inlet BC

| 확인 항목 | 결과 |
|-----------|------|
| Z=0 적용 조건 | `if self.solid[i, j, 0] == 0` 일 때만 `feq(1.0, u_in)` 적용 |
| 유체 전용 적용 | **예** |

**결론:** inlet은 **유체 셀에만** u_lb_in 적용.

## 9. 결론

- **Phase 1:** 코드 리뷰로 p_scale·ΔP 경로, 유체 전용 ρ 평균, inlet 유체 전용 적용 모두 정상. 1/3 비율의 코드상 원인은 미특정.
- **L2 재실행:** 7k 스텝 수렴, ΔP_sim=0.032008 Pa, ΔP_theory=0.096308 Pa, 비율 0.332. 판정 FAIL(ΔP 오차 66.77%). Q, clips, CV, Δρ 기준은 충족.
- **Phase 2 VTR·시각화:** 7k 스텝, 1k 간격 VTR 저장 완료. plan §3.2 항목별 PNG 5종 생성. 산출물은 `results/l2a_vti/`, `results/plan23v_vis/` 에 구조화. ParaView에서 `results/l2a_vti/l2a_diag.vtr` 또는 `l2a_diag_step7000.vtr` 사용.
- **Phase 3 (Gyroid 3-g):** 본 보고서에서 제외.
